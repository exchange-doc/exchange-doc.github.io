接入流程

基本信息

Path： /doc/readme

Method： GET

接口描述：

开放平台 1.0
接入方:商家
获取商家在平台对应的唯一身份表示 appKey 及密钥。通过 appKey 可访问交易所开放的资源。
目前开放功能:
n oauth 登录授权; n 下单;
n 支付;
n 退款;n 场外订单流程操作;
通过开放平台可以使用交易所内的账户进行 oauth 登录授权;
授权后可进行一系列的操作如:下单、支付等;
l 在授权后可以通过平台进行下单、支付、，在客户端中接入交易所的支付功能。最终资金流转从用户到商家的交易所账户内。
l 添加场外交易接口，获取场外流动性、对场外广告进行下单、交易等操作; l 其他对账等功能;
服务商接入
与客户经理发起申请->提交商家材料及信息-->审核-->获得 appKey 及密钥->调用交易所
开放资源。
需要商家提供的信息
l 商家域名，域名必须真实有效，oauth 跳转时会进行效验。
l 商家在交易所内的 userid,userId 必须是自己的，涉及到支付后的收款。快速接入
l 第一步:创建应用并获取 appKey; l 第二步:配置密钥;
l 第三步:完善签名生成方法;
l 第四部:访问开放的资源;签名规则:
· 功能描述: 开放平台对外统一网关，提供对资源访问的鉴权等功能。
· 签名规则:所有的资源都有效验功能，通常情况下，大部分接口都需要进
行防篡改签名效验;

  1. 生成签名的规则: 请求参数按照字典排序，然后以 keyvalue 的形式拼
    接成字符串 string;最后 sign=MD5(sortParam+secretKey)。每个
    商户都有自己的 appkey 和 secret;
  2. 注意:如果请求参数中value为NULL的情况，则在拼接字符串时不计
    入签名字符串。
  3. 所有的签名效验都是在网关层进行的，后面的服务调用将不在有签名规
    则。
  4. 例如:
    参数如下:
    {
    'symbol': 'ltcbtc',
    'appKey': '0816016bb06417f50327e2b557d39aaa',
    'sign': 'a169740d0588141ef70b71cf11ff8bf3',
    'time': '1522055680'
    }
    拼接完成后:
    string=appKey0816016bb06417f50327e2b557d39aaasymbolltcbt
    ctime1522055680
    sign = MD5(string+secretKey) =
    MD5(appKey0816016bb06417f50327e2b557d39aaasymbolltcbtcti
    me1522055680xxxxxxxxxxxxxxxxx)
    最终:sign=48105baf432a6626051a72fb160b617e
