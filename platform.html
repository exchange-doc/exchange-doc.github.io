第一步，OAuth授权登录页

基本信息

Path： /login.html

Method： GET

接口描述：

最后登录页拼接完成后为:
https://platform.xxx.com/login.html?appKey=xxx&redirectUrl=xxx&state=xxx

请求参数

Query

参数名称 是否必须 示例 备注
appKey 是 shangjia 商家key
redirectUrl 是 https://www.chaindown.com 授权登录成功后跳转页面
state 否 1 商家传入状态字段

返回数据

名称类型是否必须默认值备注其他信息

第二步，使用code换取token、openId接口

基本信息

Path： /chainup/open/auth/token

Method： POST

接口描述：

请求参数

Headers

参数名称 参数值 是否必须 示例 备注
Content-Type application/x-www-form-urlencoded 是

Body

参数名称 参数类型 是否必须 示例 备注
appKey text 是 商家唯一key
code text 是 用户登录授权时返回的code
sign text 是 MD5(Sort(Param) + secret)

返回数据

名称类型是否必须默认值备注其他信息
codenumber非必须返回code
msgstring非必须成功或失败提示信息
dataobject非必须
├─ openIdstring非必须该授权用户所属该商户下的唯一openId
├─ expireInstring非必须token过期时间，秒
├─ refreshTokenstring非必须刷新token所需的token
├─ tokenstring非必须授权的token

支付

第一步，创建支付订单

基本信息

Path： /chainup/open/opay/createThirdOrder

Method： POST

接口描述：

请求参数

Headers

参数名称 参数值 是否必须 示例 备注
Content-Type application/x-www-form-urlencoded 是

Body

参数名称 参数类型 是否必须 示例 备注
appKey text 是 商家唯一编号
payCoinSymbol text 是 支付币种，大写如BTCappKey
openId text 是 付款用户的唯一id
userId text 是 付款用户uid(与openid必须存在一个)
returnPage text 是 前台跳转商家的页面
notifyPage text 是 后台通知商家的地址
appOrderId text 是 商家端订单id
orderSceneType text 是 订单的场景类型，数字枚举；
orderAmount text 是 支付金额
token text 是 授权的token
sign text 是 MD5(Sort(Param) + secret)
goods text 否 商品描述信息，可不传，传入必须为合法的json格式，否则订单创建失败 ；json中的子字段为

name : 商品名称
desc:商品描述
例：[{"name":"商品1","desc":"商品信息123balabala"},{"name":"商品2","desc":"商品信息23balabala"},{"name":"商品3","desc":"商品信息33balabala"}] |

返回数据

名称类型是否必须默认值备注其他信息
codenumber非必须
msgstring非必须
dataobject非必须
├─ signstring非必须
├─ orderNumstring非必须

第三步，支付结果回调接口

基本信息

Path： /notifyPage

Method： POST

接口描述：

该接口URL由对接商户提供，第一步创建订单时传出的参数：notifyPage
支付完成后，支付结果会通过notifyPage对应的URL进行通知

请求参数

Headers

参数名称 参数值 是否必须 示例 备注
Content-Type application/json 是

Body

名称类型是否必须默认值备注其他信息
appKeystring必须商家唯一key
outOrderIdstring必须商家端订单编号
orderNumstring必须开放平台订单编号
orderStatusstring必须订单状态
payAmountstring必须订单金额
returnUrlstring必须同步通知页面(下单接口传入)
signstring必须

返回数据

名称类型是否必须默认值备注其他信息
datastring必须客户端收到请求后返回success，代表已收到请求

├─ orderAmountstring非必须订单金额
├─ payCoinSymbolstring非必须支付币种
├─ ctimestring非必须
├─ appKeystring非必须
├─ showPrecisionstring非必须显示的精度

商家日贴对账单

基本信息

Path： /chainup/open/opay/merchantBillingDay

Method： POST

接口描述：

请求参数

Headers

参数名称 参数值 是否必须 示例 备注
Content-Type application/x-www-form-urlencoded 是

Body

参数名称 参数类型 是否必须 示例 备注
appKey text 是
orderType text 是 订单类型（1=支付订单；11=划转订单），不传查询所有订单
day text 是 统计的日期，格式：yyyy-MM-dd，不符合格式参数异常；不传查询当日订单
pageNum text 是 查询第几页，同分页逻辑；不传查所有
pageSize text 是 每页大小，同分页逻辑；不传查2000（最大为2000，数据超过2000请分页查询）
sign text 是

返回数据

名称类型是否必须默认值备注其他信息
codestring非必须
msgstring非必须
dataobject非必须
├─ countnumber非必须当日记录总数；
├─ listobject []非必须

item 类型: object

├─ merchantUserIdnumber非必须商家收款用户id；
├─ appOrderIdstring非必须商家端订单id；
├─ payCoinSymbolstring非必须支付使用币种；
├─ orderAmountnumber非必须订单操作数量；
├─ orderTypenumber非必须订单类型(1、支付订单;11、划转(退款)订单)
├─ userIdnumber非必须付款用户id；付款用户id；
├─ openIdstring非必须付款用户openid；
├─ createTimestring非必须下单时间，到毫秒；
├─ payTimestring非必须实际支付完成时间，到毫秒；
